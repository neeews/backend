# errlog — Notion 오류 기록

## 목적

작업 중 발생한 오류를 Notion **"⚠️ 생긴 오류"** 페이지(ID: `38a7aff1-25c3-80b8-8860-cdffe4a14448`)에 자동 기록한다.
오류 하나당 하위 페이지 하나를 생성한다.

---

## 실행 방법

`mcp__claude_ai_Notion__notion-create-pages`를 아래 형식으로 호출한다.

```json
{
  "parent": { "type": "page_id", "page_id": "38a7aff1-25c3-80b8-8860-cdffe4a14448" },
  "pages": [{
    "properties": { "title": "오류 제목" },
    "content": "본문 내용 (아래 구조 참고)"
  }]
}
```

---

## 페이지 제목 규칙

오류의 핵심을 한 줄로 설명한다. 예시:

- `ZDnet 이미지 미추출 — cached_image_path "none" 고착`
- `rome-modules 의존성 누락으로 Docker 빌드 실패`
- `MySQL bit(1) 컬럼 터미널 표시 버그`
- `BCrypt 해시 shell escape 오류로 비밀번호 업데이트 실패`

---

## 본문 구조 (필수)

아래 4개 섹션을 순서대로 포함한다.

```
**발생 시점:** YYYY-MM-DD

**증상**
- (사용자가 발견한 현상, 에러 메시지, 잘못된 결과 등)

**원인**
- (근본 원인, 관련 코드/설정/DB 상태 설명 포함)

**해결**
- (적용한 해결 방법, 변경한 파일/쿼리/설정 등)
```

---

## 기록 기준

기록 **O:**
- 빌드 실패 (컴파일 오류, 의존성 누락)
- 서버 오류 (500, 403, NullPointerException 등)
- 설정 오류 (DB 연결, 환경변수, Docker)
- DB 상태 이상 (예상과 다른 값, 고착 상태)
- 이미지/파일 처리 실패

기록 **X:**
- 단순 오탈자 수정
- 스타일/포매팅 변경
- 아직 미해결인 문제 (해결 후 기록)

---

## 실행 순서

1. 사용자가 `/errlog` 를 입력하거나, 대화 중 오류가 해결된 시점에 호출한다.
2. 이번 작업에서 해결한 오류 목록을 파악한다.
3. 오류가 여러 개면 `pages` 배열에 여러 항목을 넣어 한 번에 생성한다.
4. 생성된 각 페이지 URL을 사용자에게 알려준다.

---

## 주의

- 아직 해결되지 않은 오류는 기록하지 않는다.
- 본문에 코드 블록이 필요하면 Notion Markdown 형식(`` ``` ``)을 사용한다.
- 발생 시점은 **절대 날짜(YYYY-MM-DD)**로 기록한다 (상대 표현 금지).
